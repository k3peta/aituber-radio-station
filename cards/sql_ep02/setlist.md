# INSERTでデータ追加
[type: talk]

まずはINSERTっす。新しいデータを追加する命令っすよ。 {まずはインサートっす。新しいデータを追加する命令っすよ。}

INSERT INTO users (name, age) VALUES ('怪談ちゃん', 16); {インサート、イントゥ、ユーザーズ、かっこ、ネーム、エイジ、かっこ閉じ、バリューズ、かっこ、かぎかっこ、怪談ちゃん、16、かっこ閉じ、セミコロン。}

INSERT INTOの後にテーブル名を書いて、 {インサート、イントゥの後にテーブル名を書いて、}
カッコの中にカラム名を並べるっす。
VALUESの後に、入れたい値を同じ順番で書くっす。 {バリューズの後に、入れたい値を同じ順番で書くっす。}

文字列はシングルクォートで囲むっす。ダブルクォートじゃないっすよ。
数字はそのまま書けるっす。簡単っすよね。

# 複数行INSERT
[type: talk]

一度にたくさんデータを入れたい時は、VALUESの後にカンマで区切るっす。 {一度にたくさんデータを入れたい時は、バリューズの後にカンマで区切るっす。}

INSERT INTO users (name, age) VALUES ('先輩', 25), ('後輩', 20); {インサート、イントゥ、ユーザーズ、バリューズ、かっこ、先輩、25、かっこ閉じ、カンマ、かっこ、後輩、20、かっこ閉じ、セミコロン。}

これで2人分のデータが一気に入るっす。
便利っすよね。大量のデータを入れる時に重宝するっす。

# UPDATEでデータ更新
[type: talk]

次はUPDATEっす。既存のデータを書き換える命令っす。 {次はアップデートっす。既存のデータを書き換える命令っす。}

UPDATE users SET age = 17 WHERE name = '怪談ちゃん'; {アップデート、ユーザーズ、セット、エイジ、イコール、17、ウェア、ネーム、イコール、怪談ちゃん、セミコロン。}

UPDATEの後にテーブル名、 {アップデートの後にテーブル名、}
SETの後に「どのカラムをどの値に変えるか」、 {セットの後に「どのカラムをどの値に変えるか」、}
WHEREで「どの行を更新するか」を指定するっす。 {ウェアで「どの行を更新するか」を指定するっす。}

# WHERE忘れの恐怖
[type: talk]

ここで怪談ちゃんから、本物の怖い話をするっす。
UPDATEでWHEREを忘れると……全員のデータが書き換わるっす。 {アップデートでウェアを忘れると……全員のデータが書き換わるっす。}

UPDATE users SET age = 99; {アップデート、ユーザーズ、セット、エイジ、イコール、99、セミコロン。}

これを実行すると、全ユーザーが99歳になるっす。
先輩も、後輩も、怪談ちゃんも、みんな99歳っす。
怖いっすよね……。WHEREは絶対忘れちゃダメっすよ。 {怖いっすよね……。ウェアは絶対忘れちゃダメっすよ。}

# DELETEでデータ削除
[type: talk]

最後はDELETEっす。データを消す命令っす。 {最後はデリートっす。データを消す命令っす。}
