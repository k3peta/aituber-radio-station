# COUNTで数える
[type: talk]

まずはCOUNTっす。データの件数を数える関数っすよ。 {まずはカウントっす。データの件数を数える関数っすよ。}

SELECT COUNT(*) FROM users; {セレクト、カウント、アスタリスク、フロム、ユーザーズ、セミコロン。}

これでusersテーブルに何件データがあるか分かるっす。 {これでユーザーズテーブルに何件データがあるか分かるっす。}
アスタリスクは「全部」って意味っす。全行を数えるっすよ。

特定のカラムを指定すると、NULLじゃない行だけ数えるっす。 {特定のカラムを指定すると、ヌルじゃない行だけ数えるっす。}

SELECT COUNT(email) FROM users; {セレクト、カウント、イーメール、フロム、ユーザーズ、セミコロン。}

メールアドレスが入ってる人だけ数えるってことっすね。

# SUMで合計
[type: talk]

次はSUMっす。数値の合計を出す関数っす。 {次はサムっす。数値の合計を出す関数っす。}

SELECT SUM(price) FROM orders; {セレクト、サム、プライス、フロム、オーダーズ、セミコロン。}

全注文の合計金額が一発で出るっす。
電卓いらずっすよ。SQLは便利っす。

# AVGで平均
[type: talk]

AVGは平均っす。アベレージの略っすよ。 {アベレージは平均っす。アベレージの略っすよ。}

SELECT AVG(age) FROM users; {セレクト、アベレージ、エイジ、フロム、ユーザーズ、セミコロン。}

ユーザーの平均年齢が出るっす。
小数点まで出てくるから、四捨五入したい時はROUND関数を使うっす。 {小数点まで出てくるから、四捨五入したい時はラウンド関数を使うっす。}

SELECT ROUND(AVG(age), 1) FROM users; {セレクト、ラウンド、アベレージ、エイジ、カンマ、1、フロム、ユーザーズ、セミコロン。}

これで小数第1位まで表示されるっす。

# MINとMAXで最小・最大
[type: talk]

MINは最小値、MAXは最大値っす。 {ミンは最小値、マックスは最大値っす。}
シンプルっすよね。

SELECT MIN(age), MAX(age) FROM users; {セレクト、ミン、エイジ、カンマ、マックス、エイジ、フロム、ユーザーズ、セミコロン。}

一番若い人と一番年上の人が分かるっす。
文字列に使うと、アルファベット順で最初と最後を返すっすよ。

# GROUP BYでグループ集計
[type: talk]

ここからが本番っす。GROUP BYっす。 {ここからが本番っす。グループバイっす。}
データをグループに分けて、グループごとに集計するっすよ。

SELECT department, COUNT(*) FROM users GROUP BY department; {セレクト、デパートメント、カンマ、カウント、アスタリスク、フロム、ユーザーズ、グループバイ、デパートメント、セミコロン。}

部署ごとに何人いるか数えるっす。
営業部3人、技術部5人、みたいな結果が出るっすよ。

集計関数とGROUP BYはセットで覚えるっす。 {集計関数とグループバイはセットで覚えるっす。}
これができると、データ分析っぽいことができるようになるっす。

# HAVINGで集計結果をフィルタ
[type: talk]

WHEREは集計前のフィルタっすけど、集計後にフィルタしたい時はHAVINGっす。 {ウェアは集計前のフィルタっすけど、集計後にフィルタしたい時はハビングっす。}

SELECT department, COUNT(*) FROM users GROUP BY department HAVING COUNT(*) >= 3; {セレクト、デパートメント、カウント、アスタリスク、フロム、ユーザーズ、グループバイ、デパートメント、ハビング、カウント、アスタリスク、大なりイコール、3、セミコロン。}

3人以上いる部署だけ表示するっす。
WHEREとHAVINGの違いは大事っすよ。 {ウェアとハビングの違いは大事っすよ。}
WHERE はグループ化の前、HAVINGはグループ化の後っす。 {ウェアはグループ化の前、ハビングはグループ化の後っす。}
