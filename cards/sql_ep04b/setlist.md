# 例1：誰が何を買ったか
[type: talk]
[float: media/slide_inner_join.png]

一番よくあるパターンっす。
ユーザー名と注文内容を一緒に見たい時。

SELECT u.name, p.product_name, o.total FROM orders o INNER JOIN users u ON o.user_id = u.id INNER JOIN products p ON o.product_id = p.id; {セレクト、ユードットネーム、ピードットプロダクトネーム、オードットトータル、フロム、オーダーズ、オー、インナージョイン、ユーザーズ、ユー、オン、オードットユーザーアイディー、イコール、ユードットアイディー、インナージョイン、プロダクツ、ピー、オン、オードットプロダクトアイディー、イコール、ピードットアイディー、セミコロン。}

3つのテーブルをつなげて「怪談ちゃんがペンを500円で買った」みたいな情報が取れるっす。

# 例2：まだ注文していないユーザー
[type: talk]
[float: media/slide_left_join.png]

LEFT JOINの出番っす。 {レフトジョインの出番っす。}

SELECT u.name FROM users u LEFT JOIN orders o ON u.id = o.user_id WHERE o.id IS NULL; {セレクト、ユードットネーム、フロム、ユーザーズ、ユー、レフトジョイン、オーダーズ、オー、オン、ユードットアイディー、イコール、オードットユーザーアイディー、ウェア、オードットアイディー、イズ、ヌル、セミコロン。}

LEFT JOINで全ユーザーを残して、注文がないユーザーはNULLになるっす。 {レフトジョインで全ユーザーを残して、注文がないユーザーはヌルになるっす。}
WHERE o.id IS NULLで「注文データがない人」だけ取れるっす。 {ウェア、オードットアイディー、イズ、ヌルで「注文データがない人」だけ取れるっす。}

これ、実務でめちゃくちゃ使うパターンっすよ。「退会しそうなユーザーを見つける」とかに使えるっす。

# 例3：ユーザーごとの注文集計
[type: talk]
[float: none]

JOINと集計関数の合わせ技っす。 {ジョインと集計関数の合わせ技っす。}

SELECT u.name, COUNT(o.id) AS 注文回数, SUM(o.total) AS 累計金額 FROM users u LEFT JOIN orders o ON u.id = o.user_id GROUP BY u.id, u.name ORDER BY 累計金額 DESC; {セレクト、ユードットネーム、カウント、オードットアイディー、アズ、注文回数、サム、オードットトータル、アズ、累計金額、フロム、ユーザーズ、ユー、レフトジョイン、オーダーズ、オー、オン、ユードットアイディー、イコール、オードットユーザーアイディー、グループバイ、ユードットアイディー、ユードットネーム、オーダーバイ、累計金額、デスク、セミコロン。}

ユーザー名を表示しつつ、注文回数と累計金額を集計するっす。
LEFT JOINだから、注文ゼロの人も「0回、0円」で表示されるっす。 {レフトジョインだから、注文ゼロの人も「0回、0円」で表示されるっす。}

# 例4：カテゴリと商品一覧
[type: talk]

カテゴリテーブルと商品テーブルを結合するっす。

SELECT c.category_name, p.product_name, p.price FROM categories c INNER JOIN products p ON c.id = p.category_id ORDER BY c.category_name, p.price; {セレクト、シードットカテゴリネーム、ピードットプロダクトネーム、ピードットプライス、フロム、カテゴリーズ、シー、インナージョイン、プロダクツ、ピー、オン、シードットアイディー、イコール、ピードットカテゴリアイディー、オーダーバイ、シードットカテゴリネーム、ピードットプライス、セミコロン。}

「食品カテゴリ：りんご100円、みかん80円」みたいに整理された一覧が出るっす。

# 例5：自分自身とJOIN
[type: talk]

ちょっと上級っすけど、同じテーブル同士のJOINもできるっす。 {ちょっと上級っすけど、同じテーブル同士のジョインもできるっす。}
セルフジョインって言うっすよ。

SELECT e.name AS 社員, m.name AS 上司 FROM employees e LEFT JOIN employees m ON e.manager_id = m.id; {セレクト、イードットネーム、アズ、社員、エムドットネーム、アズ、上司、フロム、エンプロイーズ、イー、レフトジョイン、エンプロイーズ、エム、オン、イードットマネージャーアイディー、イコール、エムドットアイディー、セミコロン。}

社員テーブルを2回使って、社員と上司の関係を出すっす。
別名をつけるから、同じテーブルでも区別できるっすよ。

# JOINを書くコツ
[type: talk]

JOINを書くコツをまとめるっす。 {ジョインを書くコツをまとめるっす。}

1つ目。まず「何のデータが欲しいか」を決めるっす。
2つ目。「そのデータはどのテーブルにあるか」を確認するっす。
3つ目。テーブル間の「共通のカラム」を見つけるっす。
4つ目。必ず別名をつけるっす。uとかoとか、短い名前でOKっすよ。 {4つ目。必ず別名をつけるっす。ユーとかオーとか、短い名前でオーケーっすよ。}
