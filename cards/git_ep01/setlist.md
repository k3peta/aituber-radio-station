# Gitって何？
[type: talk]

Gitはバージョン管理ツールっす。 {ギットはバージョン管理ツールっす。}
簡単に言うと「ファイルの変更履歴を記録してくれるやつ」っす。

たとえば先輩、レポートを書いてて「レポート最終版.docx」「レポート最終版2.docx」「レポート本当の最終版.docx」 {たとえば先輩、レポートを書いてて「レポート最終版」「レポート最終版2」「レポート本当の最終版」}
ってファイルが増えたことないっすか？

Gitを使えば、1つのファイルで全部の変更履歴が残るっす。 {ギットを使えば、1つのファイルで全部の変更履歴が残るっす。}
いつ、誰が、何を変えたか。全部分かるっすよ。

# なぜGitが必要？
[type: talk]

Gitが便利な場面はたくさんあるっす。 {ギットが便利な場面はたくさんあるっす。}

1つ目。間違えても元に戻せるっす。
セーブポイントみたいなもんっす。ゲームのセーブと同じっすよ。

2つ目。チームで同じファイルを同時に編集できるっす。
誰かの変更を上書きしちゃう心配がないっす。

3つ目。「いつ何を変えたか」が全部記録に残るっす。
バグが出た時に「あの変更が原因だ！」って特定できるっす。

# GitとGitHubの違い
[type: talk]

ここで大事な話っす。GitとGitHubは別物っすよ。 {ここで大事な話っす。ギットとギットハブは別物っすよ。}

Gitはツールっす。パソコンにインストールして使うソフトっす。 {ギットはツールっす。パソコンにインストールして使うソフトっす。}
GitHubはサービスっす。インターネット上でGitのデータを保管・共有できるサイトっす。 {ギットハブはサービスっす。インターネット上でギットのデータを保管・共有できるサイトっす。}

つまり、Gitで記録して、GitHubで共有する。 {つまり、ギットで記録して、ギットハブで共有する。}
この2つをセットで使うのが一般的っす。

# リポジトリって何？
[type: talk]

Gitの世界で一番大事な言葉が「リポジトリ」っす。 {ギットの世界で一番大事な言葉が「リポジトリ」っす。}
略して「リポ」って呼ぶこともあるっす。

リポジトリは「プロジェクトの保管庫」っす。
ファイルと、その変更履歴が全部入ってるフォルダのことっすよ。

パソコンにあるのが「ローカルリポジトリ」。
GitHubにあるのが「リモートリポジトリ」っす。 {ギットハブにあるのが「リモートリポジトリ」っす。}

# 基本の3ステップ
[type: talk]

Gitの基本操作は3ステップっす。 {ギットの基本操作は3ステップっす。}

ステップ1、変更する。ファイルを編集するっす。普通に作業するだけっす。

ステップ2、ステージングする。「この変更を記録しますよ」って準備するっす。
コマンドはgit addっす。 {コマンドはギット、アドっす。}

ステップ3、コミットする。「記録しました！」って確定するっす。
コマンドはgit commitっす。 {コマンドはギット、コミットっす。}

変更する、ステージする、コミットする。この3つの繰り返しっす。

# コミットメッセージ
[type: talk]

コミットする時にはメッセージをつけるっす。
「何を変えたか」を書くんっすよ。

git commit -m "ログイン画面を追加" {ギット、コミット、マイナスエム、「ログイン画面を追加」}

このメッセージが後から見た時にめちゃくちゃ大事っす。
「修正した」だけだと、後で見た時に何を修正したか分からないっすからね。

# GitHubにアップロード
[type: talk]

ローカルで記録したコミットをGitHubに送ることを「プッシュ」って言うっす。 {ローカルで記録したコミットをギットハブに送ることを「プッシュ」って言うっす。}

git push {ギット、プッシュ}

これだけっす。
逆にGitHubからダウンロードするのは「プル」っす。 {逆にギットハブからダウンロードするのは「プル」っす。}

git pull {ギット、プル}

プッシュで送る、プルで受け取る。シンプルっすよね。
