# お店の売上を分析する
[type: talk]
[float: media/slide_aggregate_examples.png]

まずはお店のデータベースで考えるっす。
ordersテーブルに注文データが入ってるっす。 {オーダーズテーブルに注文データが入ってるっす。}

今月の売上合計を知りたい！

SELECT SUM(total) FROM orders WHERE order_date >= '2025-03-01'; {セレクト、サム、トータル、フロム、オーダーズ、ウェア、オーダーデイト、大なりイコール、2025年3月1日、セミコロン。}

WHEREで期間を絞って、SUMで合計するっす。 {ウェアで期間を絞って、サムで合計するっす。}
これだけで「今月いくら売れたか」が分かるっす。便利っすよね。

# 注文数と平均単価
[type: talk]
[float: none]

次に、注文件数と平均単価を同時に出すっす。

SELECT COUNT(*) AS 注文件数, AVG(total) AS 平均単価 FROM orders; {セレクト、カウント、アスタリスク、アズ、注文件数、カンマ、アベレージ、トータル、アズ、平均単価、フロム、オーダーズ、セミコロン。}

ASは別名をつける機能っす。結果の列名を分かりやすくできるっすよ。 {アズは別名をつける機能っす。結果の列名を分かりやすくできるっすよ。}
「100件、平均1500円」みたいに出るっす。

# 商品カテゴリ別の売上
[type: talk]

GROUP BYの出番っす。カテゴリごとの売上を見るっすよ。 {グループバイの出番っす。カテゴリごとの売上を見るっすよ。}

SELECT category, SUM(total) AS 売上合計, COUNT(*) AS 注文数 FROM orders GROUP BY category ORDER BY 売上合計 DESC; {セレクト、カテゴリ、サム、トータル、アズ、売上合計、カウント、アスタリスク、アズ、注文数、フロム、オーダーズ、グループバイ、カテゴリ、オーダーバイ、売上合計、デスク、セミコロン。}

ORDER BY DESCで売上が多い順に並ぶっす。 {オーダーバイ、デスクで売上が多い順に並ぶっす。}
「食品が一番売れてて、次が日用品」みたいな分析ができるっす。

# 月ごとの売上推移
[type: talk]

月ごとの売上を見れば、トレンドが分かるっす。

SELECT strftime('%Y-%m', order_date) AS 月, SUM(total) AS 売上 FROM orders GROUP BY 月 ORDER BY 月; {セレクト、ストリングフォーマットタイム、年月、オーダーデイト、アズ、月、サム、トータル、アズ、売上、フロム、オーダーズ、グループバイ、月、オーダーバイ、月、セミコロン。}

strftimeは日付をフォーマットする関数っす。 {ストリングフォーマットタイムは日付をフォーマットする関数っす。}
これでSQLiteで月別集計ができるっすよ。 {これでエスキューライトで月別集計ができるっすよ。}

# 売上が多い商品トップ5
[type: talk]

GROUP BYとORDER BYとLIMITを組み合わせるっす。 {グループバイとオーダーバイとリミットを組み合わせるっす。}

SELECT product_name, SUM(quantity) AS 販売数 FROM order_items GROUP BY product_name ORDER BY 販売数 DESC LIMIT 5; {セレクト、プロダクトネーム、サム、クオンティティ、アズ、販売数、フロム、オーダーアイテムズ、グループバイ、プロダクトネーム、オーダーバイ、販売数、デスク、リミット、5、セミコロン。}

LIMITで上位5件だけ取るっす。 {リミットで上位5件だけ取るっす。}
ランキングを作りたい時にめちゃくちゃ使うっすよ。

# HAVINGで条件付き集計
[type: talk]

「3件以上注文してくれたお得意様」を見つけるっす。

SELECT user_id, COUNT(*) AS 注文回数, SUM(total) AS 累計金額 FROM orders GROUP BY user_id HAVING COUNT(*) >= 3 ORDER BY 累計金額 DESC; {セレクト、ユーザーアイディー、カウント、アスタリスク、アズ、注文回数、サム、トータル、アズ、累計金額、フロム、オーダーズ、グループバイ、ユーザーアイディー、ハビング、カウント、アスタリスク、大なりイコール、3、オーダーバイ、累計金額、デスク、セミコロン。}

HAVINGは集計した後のフィルタっす。 {ハビングは集計した後のフィルタっす。}
WHEREは集計前、HAVINGは集計後。この違いを覚えてほしいっす。 {ウェアは集計前、ハビングは集計後。この違いを覚えてほしいっす。}

# 今日のまとめ
[type: talk]

はいっ、特別回のまとめっす。

集計関数の使いどころは、
1、SUMで売上合計を出す。 {1、サムで売上合計を出す。}
2、AVGで平均単価を出す。 {2、アベレージで平均単価を出す。}
3、GROUP BYでカテゴリ別・月別に分類。 {3、グループバイでカテゴリ別・月別に分類。}
4、ORDER BYとLIMITでランキング。 {4、オーダーバイとリミットでランキング。}
5、HAVINGで集計結果をフィルタ。 {5、ハビングで集計結果をフィルタ。}
6、ASで結果に分かりやすい名前をつける。 {6、アズで結果に分かりやすい名前をつける。}

SQLだけでかなりのデータ分析ができるっす！ {エスキューエルだけでかなりのデータ分析ができるっす！}
