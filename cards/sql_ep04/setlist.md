# なぜJOINが必要？
[type: talk]

例えば、usersテーブルにユーザー情報があって、ordersテーブルに注文情報があるとするっす。 {例えば、ユーザーズテーブルにユーザー情報があって、オーダーズテーブルに注文情報があるとするっす。}
「誰が何を注文したか」を知りたい時、両方のテーブルを見ないといけないっすよね。
それを1回のSQLで取れるのがJOINっす。 {それを1回のエスキューエルで取れるのがジョインっす。}

# INNER JOIN
[type: talk]
[float: media/slide_inner_join.png]

まずはINNER JOINっす。一番基本のやつっすよ。 {まずはインナージョインっす。一番基本のやつっすよ。}

SELECT users.name, orders.total FROM users INNER JOIN orders ON users.id = orders.user_id; {セレクト、ユーザーズドットネーム、カンマ、オーダーズドットトータル、フロム、ユーザーズ、インナージョイン、オーダーズ、オン、ユーザーズドットアイディー、イコール、オーダーズドットユーザーアイディー、セミコロン。}

INNER JOINは、両方のテーブルにマッチするデータだけ取るっす。 {インナージョインは、両方のテーブルにマッチするデータだけ取るっす。}
注文してないユーザーは結果に出ないっすよ。

ONの後ろに「どのカラムで結合するか」を書くっす。 {オンの後ろに「どのカラムで結合するか」を書くっす。}
ここがJOINのキモっす。 {ここがジョインのキモっす。}

# テーブルに別名をつける
[type: talk]

テーブル名が長いと大変なので、別名をつけられるっす。エイリアスって言うっすよ。

SELECT u.name, o.total FROM users u INNER JOIN orders o ON u.id = o.user_id; {セレクト、ユードットネーム、カンマ、オードットトータル、フロム、ユーザーズ、ユー、インナージョイン、オーダーズ、オー、オン、ユードットアイディー、イコール、オードットユーザーアイディー、セミコロン。}

usersをu、ordersをoにしたっす。 {ユーザーズをユー、オーダーズをオーにしたっす。}
短くてスッキリっすよね。実務ではほぼ必ず別名を使うっす。

# LEFT JOIN
[type: talk]
[float: media/slide_left_join.png]

次はLEFT JOINっす。 {次はレフトジョインっす。}

SELECT u.name, o.total FROM users u LEFT JOIN orders o ON u.id = o.user_id; {セレクト、ユードットネーム、カンマ、オードットトータル、フロム、ユーザーズ、ユー、レフトジョイン、オーダーズ、オー、オン、ユードットアイディー、イコール、オードットユーザーアイディー、セミコロン。}

LEFT JOINは、左のテーブルは全部残すっす。 {レフトジョインは、左のテーブルは全部残すっす。}
注文してないユーザーも結果に出るっす。その場合、注文データはNULLになるっす。 {注文してないユーザーも結果に出るっす。その場合、注文データはヌルになるっす。}

「全ユーザーの注文状況を見たい」って時に使うっすよ。

# RIGHT JOIN
[type: talk]
[float: none]

RIGHT JOINはLEFT JOINの逆っす。右のテーブルを全部残すっす。 {ライトジョインはレフトジョインの逆っす。右のテーブルを全部残すっす。}
でも実務ではほとんど使わないっす。
LEFT JOINでテーブルの順番を入れ替えれば同じことができるっすからね。 {レフトジョインでテーブルの順番を入れ替えれば同じことができるっすからね。}

だからLEFT JOINをしっかり覚えておけば大丈夫っす。 {だからレフトジョインをしっかり覚えておけば大丈夫っす。}

# 複数テーブルのJOIN
[type: talk]

JOINは2つだけじゃなく、3つ以上のテーブルも結合できるっす。 {ジョインは2つだけじゃなく、3つ以上のテーブルも結合できるっす。}

SELECT u.name, o.total, p.product_name FROM users u INNER JOIN orders o ON u.id = o.user_id INNER JOIN products p ON o.product_id = p.id; {セレクト、ユードットネーム、オードットトータル、ピードットプロダクトネーム、フロム、ユーザーズ、ユー、インナージョイン、オーダーズ、オー、オン、ユードットアイディー、イコール、オードットユーザーアイディー、インナージョイン、プロダクツ、ピー、オン、オードットプロダクトアイディー、イコール、ピードットアイディー、セミコロン。}

ユーザー・注文・商品の3テーブルを結合して「誰が何を買ったか」が分かるっす。
