# データベースとは
[type: talk]

データベースって、ざっくり言うと「整理された情報の集まり」っす。
先輩のスマホの連絡先、あれもデータベースみたいなもんっす。
名前、電話番号、メールアドレスが、きれいに並んでるっすよね。

お店の在庫管理とか、Twitterのツイートとか、
Amazonの商品情報とか、全部データベースに入ってるっす。
世の中のアプリは、ほぼ全部データベースを使ってるっすよ。

# SQLとは
[type: talk]

で、そのデータベースに話しかける言葉がSQLっす。
エスキューエルって読むっす。シークルって読む人もいるっすけどね。

SQLを使うと、データベースに対して、
「このデータちょうだい」とか、
「新しいデータ追加して」とか、
「このデータ消して」とか、お願いできるっす。

プログラミング言語っていうより、データベース専用の命令文って感じっす。

# テーブルって何
[type: talk]

データベースの中には「テーブル」っていうのがあるっす。
テーブルっていうのは、エクセルの表みたいなもんっす。

たとえば「ユーザー」テーブルだったら、
横の行が一人分のデータっす。
縦の列が、名前、年齢、メールアドレスとかの項目っす。

1行のデータのことを「レコード」って呼ぶっす。
1列の項目のことを「カラム」って呼ぶっす。
この2つは覚えておいてほしいっす。レコードとカラムっす。

# はじめてのSELECT
[type: talk]

じゃあ、最初のSQLを書いてみるっす。
データを取得する命令、SELECTっす。セレクトっす。

SELECT アスタリスク FROM users セミコロン。

これで、usersテーブルの全データが取れるっす。
アスタリスクは「全部ちょうだい」って意味っす。
FROMは「どのテーブルから」っす。

簡単っすよね？英語の文章みたいに読めるっす。
「全部を選んで、usersから」っす。

# WHERE で絞り込み
[type: talk]

全部取るのもいいっすけど、絞り込みたい時もあるっすよね。
そういう時はWHEREを使うっす。ウェアっす。

SELECT アスタリスク FROM users WHERE age イコールイコール 25 セミコロン。

……あ、ごめんなさい。SQLではイコールは1つっす。
プログラミングと違って、イコール1個で比較っす。

SELECT アスタリスク FROM users WHERE age イコール 25 セミコロン。

これで25歳のユーザーだけ取れるっす。
先輩の年齢で絞り込んでみてほしいっす。……年齢は聞かないっすよ？ふふ。

# ORDERBYで並べ替え
[type: talk]

取ったデータを並べ替えたい時は、ORDER BYっす。

SELECT アスタリスク FROM users ORDER BY age セミコロン。

これで年齢順に並ぶっす。
DESC って付けると逆順っす。大きい方からっす。

SELECT アスタリスク FROM users ORDER BY age DESC セミコロン。

ね。英語っぽく読めるっすよね。
データを選んで、usersから、年齢で並べて、降順でっす。
