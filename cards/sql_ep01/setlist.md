# データベースとは
[type: talk]

データベースって、ざっくり言うと「整理された情報の集まり」っす。
先輩のスマホの連絡先、あれもデータベースみたいなもんっす。
名前、電話番号、メールアドレスが、きれいに並んでるっすよね。

お店の在庫管理とか、Twitterのツイートとか、全部データベースに入ってるっす。 {お店の在庫管理とか、ツイッターのツイートとか、全部データベースに入ってるっす。}
世の中のアプリは、ほぼ全部データベースを使ってるっすよ。

# SQLとは
[type: talk]

で、そのデータベースに話しかける言葉がSQLっす。 {で、そのデータベースに話しかける言葉がエスキューエルっす。}
シークルって読む人もいるっすけどね。

SQLを使うと、データベースに対して、 {エスキューエルを使うと、データベースに対して、}
「このデータちょうだい」とか、
「新しいデータ追加して」とか、
「このデータ消して」とか、お願いできるっす。

プログラミング言語っていうより、データベース専用の命令文って感じっす。

# テーブルって何
[type: talk]

データベースの中には「テーブル」っていうのがあるっす。
テーブルっていうのは、Excelの表みたいなもんっす。 {テーブルっていうのは、エクセルの表みたいなもんっす。}

たとえば「users」テーブルだったら、 {たとえば「ユーザーズ」テーブルだったら、}
横の行が一人分のデータっす。
縦の列が、name、age、emailとかの項目っす。 {縦の列が、ネーム、エイジ、イーメールとかの項目っす。}

1行のデータのことを「レコード」って呼ぶっす。
1列の項目のことを「カラム」って呼ぶっす。
この2つは覚えておいてほしいっす。レコードとカラムっす。

# はじめてのSELECT
[type: talk]
[float: media/slide_select.png]

じゃあ、この図を見てほしいっす。
SELECT文の使い方がまとまってるっす。 {セレクト文の使い方がまとまってるっす。}
データを取得する命令、SELECTっす。 {データを取得する命令、セレクトっす。}

SELECT * FROM users; {セレクト、アスタリスク、フロム、ユーザーズ、セミコロン。}

これで、usersテーブルの全データが取れるっす。 {これで、ユーザーズテーブルの全データが取れるっす。}
*（アスタリスク）は「全部ちょうだい」って意味っす。
FROMは「どのテーブルから」っす。 {フロムは「どのテーブルから」っす。}

簡単っすよね？英語の文章みたいに読めるっす。
「全部を選んで、usersから」っす。 {「全部を選んで、ユーザーズから」っす。}

# WHEREで絞り込み
[type: talk]

全部取るのもいいっすけど、絞り込みたい時もあるっすよね。
そういう時はWHEREを使うっす。 {そういう時はウェアを使うっす。}

SELECT * FROM users WHERE age = 25; {セレクト、アスタリスク、フロム、ユーザーズ、ウェア、エイジ、イコール、25、セミコロン。}

SQLではイコールは1つっす。 {エスキューエルではイコールは1つっす。}
プログラミングと違って、=（イコール）1個で比較っすよ。

これで25歳のユーザーだけ取れるっす。
先輩の年齢で絞り込んでみてほしいっす。……年齢は聞かないっすよ？ふふ。

# ORDER BYで並べ替え
[type: talk]

取ったデータを並べ替えたい時は、ORDER BYっす。 {取ったデータを並べ替えたい時は、オーダーバイっす。}

SELECT * FROM users ORDER BY age; {セレクト、アスタリスク、フロム、ユーザーズ、オーダーバイ、エイジ、セミコロン。}

これで年齢順に並ぶっす。
DESCって付けると逆順っす。大きい方からっす。 {デスクって付けると逆順っす。大きい方からっす。}

SELECT * FROM users ORDER BY age DESC; {セレクト、アスタリスク、フロム、ユーザーズ、オーダーバイ、エイジ、デスク、セミコロン。}

ね。英語っぽく読めるっすよね。
データを選んで、usersから、ageで並べて、降順でっす。 {データを選んで、ユーザーズから、エイジで並べて、降順でっす。}
