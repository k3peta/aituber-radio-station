# Gitのインストール確認
[type: talk]

まず、Gitが入ってるか確認するっす。 {まず、ギットが入ってるか確認するっす。}
ターミナルを開いて、こう打つっす。

git --version {ギット、マイナスマイナス、バージョン。}

git version 2.39.0みたいに出たらOKっす。 {ギット、バージョン、2.39.0みたいに出たらオーケーっす。}
Macなら最初から入ってることが多いっすよ。 {マックなら最初から入ってることが多いっすよ。}

# 最初の設定
[type: talk]

Gitを使う前に名前とメールアドレスを設定するっす。 {ギットを使う前に名前とメールアドレスを設定するっす。}

git config --global user.name "怪談ちゃん" {ギット、コンフィグ、マイナスマイナス、グローバル、ユーザードットネーム、怪談ちゃん}

git config --global user.email "kaidan@example.com" {ギット、コンフィグ、グローバル、ユーザードットイーメール}

これはコミットの時に「誰が変更したか」を記録するために必要っす。
1回だけ設定すればOKっすよ。 {1回だけ設定すればオーケーっすよ。}

# リポジトリを作る
[type: talk]

じゃあ実際にリポジトリを作るっす。
まず適当なフォルダを作って、その中に移動するっす。

mkdir my-project {エムケーディア、マイプロジェクト。}
cd my-project {シーディー、マイプロジェクト。}

そしてGitリポジトリとして初期化するっす。 {そしてギットリポジトリとして初期化するっす。}

git init {ギット、イニット。}

これで .git っていう隠しフォルダが作られて、バージョン管理が始まるっす。 {これで、ドットギットっていう隠しフォルダが作られて、バージョン管理が始まるっす。}

# ファイルを作って記録する
[type: talk]

試しにファイルを作ってみるっす。

echo "Hello Git" > hello.txt {エコー、ハローギット、リダイレクト、ハロー、テキスト。}

このファイルをGitに記録するっす。まずステージングっす。 {このファイルをギットに記録するっす。まずステージングっす。}

git add hello.txt {ギット、アド、ハロー、テキスト。}

そしてコミットっす。

git commit -m "最初のコミット" {ギット、コミット、マイナスエム、最初のコミット。}

やったっす！これで最初のセーブポイントが作られたっす。

# 変更を記録する
[type: talk]

じゃあファイルを変更して、もう1回コミットしてみるっす。

hello.txtに「こんにちは」を追加したとするっす。 {ハロー、テキストに「こんにちは」を追加したとするっす。}

変更状態を確認するコマンドがあるっす。

git status {ギット、ステータス。}

modified: hello.txtって出るっす。 {モディファイド、ハロー、テキストって出るっす。}
「このファイルが変更されたよ」って教えてくれるっすよ。

git add hello.txt {ギット、アド、ハロー、テキスト。}
git commit -m "こんにちはを追加" {ギット、コミット、マイナスエム、こんにちはを追加。}

これで2つ目のセーブポイントっす。

# 履歴を見る
[type: talk]

コミットの履歴を見るにはgit logっす。 {コミットの履歴を見るにはギット、ログっす。}

git log {ギット、ログ。}

こんにちはを追加、最初のコミット、って順番に出るっす。
誰が、いつ、何をしたかが全部分かるっすよ。

短く表示したい時はワンラインオプションっす。

git log --oneline {ギット、ログ、マイナスマイナス、ワンライン。}

1コミット1行でスッキリ見えるっす。

# GitHubに送る
[type: talk]

ローカルで作ったリポジトリをGitHubに送るっす。 {ローカルで作ったリポジトリをギットハブに送るっす。}

まずGitHubでリポジトリを作るっす。サイトの右上のプラスボタンから New repository っすよ。 {まずギットハブでリポジトリを作るっす。サイトの右上のプラスボタンから、ニューリポジトリっすよ。}

作ったら、ターミナルでリモートを登録するっす。

git remote add origin https://github.com/自分のID/my-project.git {ギット、リモート、アド、オリジン、自分のリポジトリのURL。}

そしてプッシュっす。

git push -u origin main {ギット、プッシュ、マイナスユー、オリジン、メイン。}

-uは最初だけ必要っす。2回目からはgit pushだけでOKっすよ。 {マイナスユーは最初だけ必要っす。2回目からはギット、プッシュだけでオーケーっすよ。}

# 今日のまとめ
[type: talk]
[float: media/slide_git_commands.png]

はいっ、第2回のまとめっす。

今日使ったコマンドは、
1、git init でリポジトリ作成。 {1、ギット、イニットでリポジトリ作成。}
2、git add でステージング。 {2、ギット、アドでステージング。}
3、git commit -m でコミット。 {3、ギット、コミット、マイナスエムでコミット。}
4、git status で状態確認。 {4、ギット、ステータスで状態確認。}
5、git log で履歴確認。 {5、ギット、ログで履歴確認。}
6、git push でGitHubに送る。 {6、ギット、プッシュでギットハブに送る。}

全部覚えなくても大丈夫っす。使ってるうちに自然に覚えるっすよ。
